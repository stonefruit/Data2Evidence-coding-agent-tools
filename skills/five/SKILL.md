# Five Whys

Use to move from symptom to root cause without jumping straight to a patch.

## Workflow

1. State the observed problem in concrete terms.
2. Ask why it happened and answer with evidence, not guesses.
3. Repeat until the root cause is actionable; five rounds is a guideline, not a quota.
4. Validate by working backward from the proposed root cause to the symptom.
5. Propose fixes that address the root cause and identify any quick mitigation separately.
